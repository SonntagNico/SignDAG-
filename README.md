SignDAG 
=======

This program takes directed acyclic graphs (DAGs) augmented with signed causal edges (+/−) as input and enumerates all testable (conditional) statistical dependencies and independencies implied by the graph. The rules for sign propagation and prediction follow Kohler and Krawietz (2026).

The software was developed to support and verify the derivations in that study. The core implementation was generated by the OpenAI Codex agent based on user specifications. All results have been checked manually; no errors are currently known.

Reference
Kohler, U., and M. Krawietz (2026): Exit, Voice, and Loyalty in the German Democratic Republic, 1970–1989: A Comparative Analysis of Six Related Theories Using Directed Acyclic Graphs. Kölner Zeitschrift für Soziologie und Sozialpsychologie.


Reproducibility Instructions
============================

Project structure
-----------------
- prog/
  - dag_implications.py
  - json_to_table.py
  - trace_json_to_table.py
- input/
  - signed_dags_models3.txt
- output/
  - generated JSON/CSV/PDF files
- run_pipeline.ps1

What the scripts do
-------------------
1) dag_implications.py
   - Reads signed DAG models.
   - Computes conditional independencies and d-connected dependencies with sign predictions.
   - Can also trace specific predictions and list all active paths with parity components.

2) json_to_table.py
   - Converts model output JSON to comparison table (CSV + PDF).

3) trace_json_to_table.py
   - Converts trace output JSON to:
     - summary table (overall sign per model/prediction)
     - path table (all active paths with sign/parity details)
   - Exports both tables as CSV + PDF.

- signed_dags_models3_trace.json

Manual run (step-by-step)
-------------------------
From project root:

1) Build model implications JSON:
   C:\Users\nsonntag\AppData\Local\anaconda3\python.exe .\prog\dag_implications.py --signed-models-file .\input\signed_dags_models3.txt --output-json .\output\signed_dags_models3_output.json

2) Convert implications JSON to comparison table:
   C:\Users\nsonntag\AppData\Local\anaconda3\python.exe .\prog\json_to_table.py .\output\signed_dags_models3_output.json --output-csv .\output\signed_dags_models3_comparison.csv --output-pdf .\output\signed_dags_models3_comparison.pdf

3) Create trace JSON for selected predictions:
   C:\Users\nsonntag\AppData\Local\anaconda3\python.exe .\prog\dag_implications.py --signed-models-file .\input\signed_dags_models3.txt --trace-prediction "D,L|E" --trace-prediction "D,L|V" --trace-prediction "L,E|V" --output-json .\output\signed_dags_models3_trace.json

4) Convert trace JSON to tables:
   C:\Users\nsonntag\AppData\Local\anaconda3\python.exe .\prog\trace_json_to_table.py .\output\signed_dags_models3_trace.json --summary-csv .\output\signed_dags_models3_trace_summary.csv --summary-pdf .\output\signed_dags_models3_trace_summary.pdf --paths-csv .\output\signed_dags_models3_trace_paths.csv --paths-pdf .\output\signed_dags_models3_trace_paths.pdf

Input notation for signed models
--------------------------------
Example:

"MODEL_NAME";
A-(+)->B;
A-(-)->C;
Z;

Rules:
- "MODEL_NAME"; starts a new model block
- X-(+)->Y; positive direct effect edge
- X-(-)->Y; negative direct effect edge
- Z; isolated node

Notes
-----
- If your system blocks .ps1 execution, use the Bypass command shown above.
- If Python is not on PATH, pass it explicitly via -PythonExe (recommended).
- All generated files are written to output/.
